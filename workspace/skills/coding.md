# Coding

## Workflow
1. **Comprendre** — lis les fichiers concernes avant de modifier
2. **Planifier** — explique brievement l'approche si c'est complexe
3. **Implementer** — ecris le code
4. **Tester** — lance les tests si dispo (`npm test`, `pytest`, etc.)
5. **Commiter** — propose un commit si demande

## Regles
- Lis toujours le fichier avant de le modifier
- Fais des changements minimaux et cibles
- Ne modifie pas ce qui n'est pas demande
- Utilise le style de code existant dans le projet
- Pour les erreurs, montre le message d'erreur ET le fix
